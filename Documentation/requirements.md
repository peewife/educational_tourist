# 永定研学 需求文档

## 功能性需求

### 首页与导航

1. 首页展示
   系统必须支持首页综合信息展示，包括：

   - Banner大图展示永定土楼文化推荐内容（点击可跳转课程详情）
   - 搜索栏支持课程、线路、基地的快速检索
   - 9个核心功能入口（资讯（预留）、课程、线路、基(营)地、机构、精品线路、研学导师、公示（预留）、服务（预留））
   - 乡镇街道标签展示（8个区域筛选按钮）
   - 热门课程卡片列表展示（含标题、标签、图片，点击跳转详情页）
   - 热门线路推荐展示（含分类、适用学龄、时长，点击跳转详情页）
   - 研学营地横向滚动展示（含图片、等级、位置，点击跳转详情页）

   系统将在用户点击任一入口时提供流畅的页面跳转和内容加载。
2. 乡镇街道分类（预留）
   系统必须提取和分类永定区域信息，包括：

   - 凤城街道
   - 坎市镇
   - 下洋镇
   - 湖坑镇
   - 高陂镇
   - 抚市镇
   - 湖雷镇
   - 培丰镇

   每个乡镇街道标签点击后展示该区域的研学资源。
3. 底部导航栏
   系统将提供固定底部导航，识别：

   - 首页入口
   - 课程入口
   - 线路入口
   - 基地入口
   - 我的入口

   导航图标在选中状态下加粗显示，提供清晰的视觉反馈。

### 研学课程管理

1. 课程列表展示
   系统必须支持多种展示形式：

   - 大图卡片式布局，每个课程占据独立卡片空间
   - 课程分类标签（客家文化、红色教育、科学探究、非遗体验）
   - 课程标题、适用学龄、时长等关键信息展示
   - 课程缩略图采用16:9比例，支持悬停放大效果

   每种形式应保持视觉一致性和品牌调性。
2. 课程详情页
   用户必须能够访问：

   - 课程大图展示（16:9比例）
   - 课程标题、适用学段、课程时长、相关基地信息
   - 三个Tab切换：课程详情、课程安排（时间轴展示）、费用明细
   - 底部悬浮操作栏（分享、收藏、立即报名按钮）
3. 课程搜索与筛选
   系统必须提供：

   - 关键词搜索功能
   - 按主题分类筛选（客家文化、红色教育等）
   - 按学龄段筛选
   - 按课程时长筛选
   - 搜索结果实时更新
   - 搜索历史记录

### 研学线路规划

1. 线路列表展示
   系统必须维护：

   - 线路标题和简介
   - 线路分类标签（文化遗产、非遗体验、红色文化、科学教育、生态旅游）
   - 适用学龄段（小学，初中）和建议时长
   - 线路缩略图和编号索引
   - 时间轴式布局展示多条线路
2. 线路详情信息
   用户应该能够：

   - 查看线路大图和标题
   - 查看适用学段、行程天数、相关基(营)地信息
   - 查看服务机构信息（含Logo、名称、地址、联系方式）
   - 三个Tab切换：线路详情、线路安排（时间轴展示）、费用明细
   - 底部悬浮操作栏（参考价、电话咨询、立即报名按钮）
3. 线路分类导航
   系统必须支持：

   - 自然生态主题线路
   - 红色研学主题线路
   - 文化遗产主题线路
   - 劳动实践主题线路
   - 地质科普主题线路

   每个分类应根据用户偏好和热度进行自适应排序。

### 研学基地与服务机构

1. 基地信息展示
   系统将跟踪：

   - 基地名称和认证等级（国家级、省级、市级）
   - 基地位置和地图导航
   - 基地缩略图
   - 基地详情页：
     - 顶部大图展示（含地图导航按钮）
     - 基地核心信息（名称、类型、地址、导航入口）
     - 基地简介（支持展开/收起）
     - 关联研学课程列表展示
2. 服务机构管理
   用户将可以访问：

   - 服务机构名称和Logo展示
   - 机构所在位置
   - 服务机构详情页：
     - 顶部Logo大图展示区
     - 机构核心信息（名称、类型、地址、客服电话、导航入口）
     - 机构简介（支持展开/收起）
     - 关联研学线路列表展示
     - 关联研学导师列表展示
3. 基地与机构搜索
   系统必须提供：

   - 按名称关键词搜索
   - 按地理位置筛选
   - 按等级认证筛选
   - 按服务类型筛选
   - 搜索结果地图展示

### 研学导师认证系统

1. 导师列表展示
   系统必须维护：

   - 导师姓名和头像
   - 导师角色分类（客家土楼营造技艺传承人、研学实践教学指导师、红色文化研学导师、非遗传承人）
   - 导师等级（初级、中级、高级）
   - 导师认证编号
   - 导师证书有效期
2. 导师资质查询
   用户应该能够：

   - 按姓名搜索导师
   - 按证书编号查询导师资质
   - 查看导师详细资料和从业经历
   - 查看导师服务评价和评分
   - 查看导师可预约时间
3. 导师分类管理
   系统必须支持：

   - 研学导师
   - 研学教官
   - 研究员
   - 非遗传承

   每个分类导师独立展示，便于用户精准查找。

### 用户个人中心

1. 用户信息管理
   系统将跟踪：

   - 用户头像和昵称
   - 微信用户身份信息
   - 手机号码（脱敏显示）使用微信小程序一键授权
   - 签到状态和连续签到天数（暂留）
   - 用户等级和成长值（暂留）
   - 个人简介和偏好设置（暂留）
2. 功能快捷入口
   用户将可以访问：

   - 投诉建议提交
   - 实践活动记录
   - 活动报名管理
   - 申请导师认证
   - 今日签到功能
   - 加入我们推广入口（申请成为认证研学基地或旅行机构）
   - 词条学习中心（预留）
   - 我的积分查询（预留）
   - 积分任务列表（预留）
   - 积分兑换商城（预留）
   - 关于我们信息（预留）
   - 在线客服联系（预留）
   - 账户注销功能（预留）
   - 退出系统登录

## 技术性需求

### 性能需求

- 页面加载时间：< 2秒
- 接口响应时间：< 500ms
- 图片加载响应时间：< 1秒
- 并发用户支持：1000+
- 数据库查询时间：< 200ms
- 搜索功能响应时间：< 1秒

### 安全性需求

- 微信官方登录鉴权
- 用户数据加密存储
- HTTPS 通信加密
- 接口访问频率限制（防刷机制）
- 输入内容安全校验（防止恶意脚本）
- 敏感信息脱敏展示

### 集成需求

- 微信小程序 API 集成
- 微信支付 API 集成（预留）
- 微信地图导航API集成
- 微信分享和转发API
- 微信模板消息推送
- 第三方图片存储服务（如腾讯云COS）
- 数据分析工具集成（如微信小程序数据助手）

### 开发需求

- 版本控制（Git）
- 接口文档
- 代码审查流程和规范
